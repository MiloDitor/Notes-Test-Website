Conquering the Command Line: Chapter 1

  • PWD \\\ pwd is the command to your home directory.
  • -P \\\ To see the actual physical path do -P on the end of pwd
  • ls \\\ ls is the command to show what files are in your directory.
  • -a \\\ To see the hidden and revealed files via .dot we would put a -a infront of ls "ls -a"
  • -l \\\ Put -l infront of ls to see more details about the files "ls -l", furthermore putting h in there gives you exact sizes of the files "ls -lh"
  • -s \\\ Using -s we can sort the files by size "ls -lhs"
  • -t \\\ Put -t to show the last modified time "ls -lt"
  • -r \\\ You can also put -r to reverse the list "ls -lr"
  • ln \\\ ln is a separate code, you would use it to bridge two files "ln a.txt b.txt"
  • info \\\ In the example above a.txt is the source file and b.txt is the target file
  • info \\\ This is called a hard link and it is the default link, this basically duplicates the file and all of its contents, this can only work as long as the files are on the same file system. Duplicating the a.txt essentially makes the b.txt through that command
  • -s \\\ Using the -s flag in ln makes it a symbolic link which means it can work in directories "ln -s a.txt b.txt"
  • info\\\ If the source file gets deleted the target new file will continue to exist independently, despite the now severed link.
  • rm \\\ rm is how you remove a file "rm test.txt"
  • touch \\\ while touch is how you create "touch test.txt"
  • cat \\\ Using cat, you can see the contents of the file, "cat test.txt"
  • -f \\\ If the suggested target link already exists you will be met with an error, you can bypass this error and overwrite the file by using a -f flag. "lm -f a.txt b.txt"
  • cd \\\ You will eventually need to transfer to another directory, to do this use the cd (change directory) flag. "cd -/Documents"
  • mkdir \\\ We can create a directory by using the mkdir command. "mkdir (name of directory)"
  • -p \\\ Using the -p flag you can create nested directories which are just subfolders in said directory. "mkdir -p a/b/c"
  • -v \\\ You can use the -v (verbose output) command which will print the results of the mkdir to the console "mkdir -v a", "mkdir: created directory 'a' ".
  • cp \\\ Copying files (cp) are different from link (ln), using cp you can copy files to directories or copy entire directories. "cp a.txt b.txt"
  • info \\\ You can also use cp to copy multiple files at once but you must end the command with a directory and the original file names will be used. "cp a.txt b.txt (directory name)"
  • -v \\\ Using -v within your cp will show the verbose output for the files. "cp -v a.txt b.txt", "a.txt -> b.txt"
  • -R \\\ To copy directories without an error use the -R "cp -Rv (direct a) (direct b)"
  • -f \\\ When overwriting a file without permission (another users file) you can use the -f flag to force overwrite. "cp -f a.txt b.txt"
  • -i \\\ When overwriting several files cp will send you a confirmation if you use the -i flag which will direct you to a y/n. "cp -i a.txt b.txt
  • info \\\ Moving files is essentially copying while removing it from its previous location, the commnd for this is mv. "mv a.txt b.txt"
  • grep \\\ use grep as a search tool "grep smile"
  • | \\\ You can use |(Redirecting Output) to add more commands on a single line, in the example shown you can see using the | you can search for a term off the previous command which directed you to the home directory. "ls -a - | grep _"
  • > \\\ Using the write operator (>) you can directly write into a file. "ls -a ~ | grep _ > underscores.txt"
  • < \\\ Using the read operator (<) you can read from the file.
